# 02. 깃허브 기초 — 내 작업을 저장·백업하기 (📗 필요할 때)

뭔가 만들기 시작하면 알면 좋은 개념입니다. 지금 당장은 안 읽어도 됩니다.

## 깃(Git)과 깃허브(GitHub)가 뭔가요?

- **깃(Git)** = 작업의 **"저장 시점"을 찍어두는 기능.** 게임의 세이브 포인트라고 생각하세요.
  잘못돼도 이전 저장 지점으로 되돌릴 수 있습니다.
- **깃허브(GitHub)** = 그 저장본을 **인터넷 창고에 보관**하는 곳. 컴퓨터가 고장나도 안전하고, 남과 공유도 됩니다.

| 용어 | 쉽게 |
| --- | --- |
| 레포(repo) | 프로젝트 하나가 담긴 폴더(창고) |
| 커밋(commit) | "지금 상태 저장" (세이브) |
| 푸시(push) | 저장본을 인터넷 창고(깃허브)로 올리기 |
| 공개/비공개(public/private) | 남이 볼 수 있냐 없냐 |

## 왜 쓰나요?
- 💾 **백업**: 컴퓨터가 망가져도 작업이 안 날아감
- ⏪ **되돌리기**: 어제 상태로 복구 가능
- 🤝 **공유**: 남에게 링크로 전달 (이 스타터도 깃허브로 배포됩니다)

## AI에게 이렇게 시키세요
직접 명령어를 외울 필요 없습니다.

> "이 폴더를 깃으로 관리하고, 지금까지 한 걸 저장(커밋)해줘."
> "내 깃허브에 **비공개** 저장소로 올려줘." (← 남에게 안 보이게 하려면 '비공개')
> "어제 저장한 상태로 되돌려줘."

## ⚠️ 주의 (01번과 연결)
깃허브에 올릴 때 **비밀키가 같이 올라가지 않게** 하세요.
AI에게 "비밀 값은 빼고 올려줘 (`.gitignore` 확인해줘)"라고 한마디 덧붙이면 됩니다.
→ 자세히는 [`01_비밀키_안전.md`](01_비밀키_안전.md)

## 한 줄 요약
> 깃허브 = 내 작업의 **인터넷 백업 창고**. "저장하고 깃허브에 올려줘"라고 AI에게 시키면 끝.
> 단, **비밀키는 빼고** 올리기.
