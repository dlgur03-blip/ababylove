# 04. API 기초 — 다른 서비스와 연결하기 (📗 필요할 때)

"API"라는 말이 자주 나오는데, 개념만 잡으면 됩니다.

## API가 뭔가요?

**프로그램끼리 대화하는 창구**입니다. 식당 비유로:
- 나(내 프로그램) = 손님
- 다른 서비스(날씨·지도·결제 등) = 주방
- **API = 주문을 받아 전달하는 종업원**

내가 "서울 날씨 알려줘"라고 API에 요청하면, 그 서비스가 답을 돌려줍니다.
덕분에 날씨·지도·번역·결제 같은 걸 **내가 안 만들고 가져다 쓸** 수 있어요.

## API 키가 왜 필요한가요?
대부분의 API는 아무나 못 쓰게 **열쇠(API 키)**를 요구합니다.
"누가 얼마나 썼는지" 확인하고 요금을 매기기 위해서요.
→ 그래서 **API 키 = 비밀** 입니다. ([`01_비밀키_안전.md`](01_비밀키_안전.md) 꼭 같이 보기)

## AI에게 이렇게 시키세요
> "○○ 서비스의 API를 연결해서 △△를 가져오게 해줘. 키를 어디서 발급받는지도 알려줘."
> "API 키는 비밀로 안전하게 넣어줘." (← 이 한마디 꼭)

- 키 **발급**은 보통 본인이 그 서비스 사이트에서 직접 받아야 합니다 — AI가 "여기서 발급받으세요"라고 안내합니다.
- 발급받은 키를 **AI에게 채팅으로 붙여넣지 말고**, "환경변수로 넣어줘"라고 시키세요.

## 한 줄 요약
> API = 다른 서비스를 **가져다 쓰는 창구**, API 키 = 그 창구의 **비밀 열쇠**.
> "○○ API 연결해줘, 키는 비밀로 넣어줘"라고 AI에게 시키면 됩니다.
