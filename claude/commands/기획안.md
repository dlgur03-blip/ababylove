# 기획안 작성

사용자의 요청을 분석하여 구조화된 기획안 MD 파일을 작성한다.
복잡하거나 여러 파일을 바꾸는 작업 전에 먼저 계획을 세우는 도구다.

## 절차

### 1. 시각 확인
- **반드시** 시스템 시각을 먼저 확인하라:
  ```bash
  date "+%Y%m%d_%H%M"
  ```
- 이 값을 파일명과 문서 내부에 사용

### 2. 현황 파악
- 프로젝트 구조 탐색 (Glob/Grep으로 관련 파일 파악)
- 기존 기획안/작업보고 확인: `Glob("*기획안.md")`, `Glob("*작업보고.md")`

### 3. 기획안 작성
- **파일명**: `YYYYMMDD_HHMM_목적요약_기획안.md`
  - 예: `20260419_1430_로그인_리팩터링_기획안.md`
- **저장 위치**: 현재 프로젝트 루트 (또는 사용자가 지정한 경로)

### 4. 문서 구조 (필수)

```markdown
# [목적 요약] 기획안

- **작성 시각**: YYYY-MM-DD HH:MM (시스템 시각)
- **목적**: 이 기획이 달성하려는 것
- **프로젝트 진행 단계**: 현재 어디까지 와 있는지
- **맥락**: 왜 지금 이 기획을 하는지

## 현황 분석
(현재 코드/구조 상태, 문제점)

## 목표
(달성해야 할 구체적 결과물)

## 작업 항목
- [ ] 작업1: 설명
- [ ] 작업2: 설명
- [ ] 작업3: 설명
(각 항목에 관련 파일/함수 명시)

## 예상 변경 사항
| 파일 | 변경 내용 | 이유 |
|------|-----------|------|
| ... | ... | ... |

## 리스크/주의사항
(기존 코드와 충돌 가능성, 의존성 등)

## 검증 방법
(작업 완료 후 어떻게 확인할 것인지)
```

### 5. 검증
- 기획안에 언급된 파일/함수/테이블이 **실제로 존재하는지** Glob/Grep으로 확인
- 수정 방향이 기존 코드 흐름과 충돌하지 않는지 확인
- 불일치 발견 시 기획안에 반영하고 사용자에게 알림

### 6. 완료 보고
- 기획안 파일 경로 출력
- 작업 항목 수와 예상 영향 범위 요약
- "/실행 기획안파일경로" 로 실행 가능함을 안내
