# 작업보고 작성

현재 세션에서 수행한 작업을 분석하여 구조화된 작업보고 MD 파일을 작성한다.
`/실행`으로 작업을 마친 뒤, 무엇을 했는지 기록으로 남기는 도구다.

## 절차

### 1. 시각 확인
- **반드시** 시스템 시각을 먼저 확인하라:
  ```bash
  date "+%Y%m%d_%H%M"
  ```

### 2. 현재 세션 분석
- 이번 세션에서 수정/생성/삭제한 파일 파악
- 시도한 접근법과 결과 (성공/실패) 정리
- 관련 기획안이 있으면 Read로 읽어서 대조

### 3. 작업보고 작성
- **파일명**: `YYYYMMDD_HHMM_목적요약_작업보고.md`
  - 예: `20260419_1530_봇연동_설정완료_작업보고.md`
- **저장 위치**: 현재 프로젝트 루트 (또는 사용자가 지정한 경로)

### 4. 문서 구조 (필수)

```markdown
# [목적 요약] 작업보고

- **작성 시각**: YYYY-MM-DD HH:MM (시스템 시각)
- **목적**: 어떤 작업이었는지
- **관련 기획안**: (있으면 파일명 기재)

## 작업 과정
1. [HH:MM] 무엇을 시도했는지 (성공/실패)
2. [HH:MM] 다음 시도 ...
(시간순으로 모든 과정을 빠짐없이 기록)

## 변경 사항 (전→후)
| 파일 | 변경 전 | 변경 후 | 이유 |
|------|---------|---------|------|
| ... | ... | ... | ... |

## 수정된 파일 목록
- `파일경로` — 신규/수정/삭제 + 한줄 설명

## 결과
- 성공/실패 여부
- 수치 변화 (있으면)
- 동작 확인 방법/결과

## 미해결 사항
- (남은 이슈, 후속 작업)
```

### 5. 규칙
- **축약 금지**: 모든 작업 과정, 시도, 변경 사항을 빠짐없이 전부 기록하라
- **전후 비교 필수**: "수정함"이 아니라 "A였던 것이 B로 변경됨"
- **시각 기록**: 각 주요 단계에 시각(HH:MM) 포함
- 관련 기획안이 있으면 기획안의 체크리스트 완료 상태도 기록
