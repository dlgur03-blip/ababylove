# 실행 모드 — 기획안 기반 자율 실행

기획안을 읽고, 작업 항목을 하나씩 실행하면서 매 단계마다 기획안을 다시 확인하고 자가 검수한다.
`/기획안`으로 만든 계획을 실제로 수행하는 도구다.

## 핵심 원칙
1. **기획안이 기준이다** — 기획안에 없는 작업은 하지 마라
2. **매 작업 완료마다 기획안 재확인** — Read로 기획안을 다시 읽고 현재 위치 파악
3. **자가 검수 후 진행** — 검수 통과해야 다음 작업으로 넘어감
4. **전후 비교 중심 체크리스트** — 뭐가 바뀌었는지를 보여줘라

## 실행 절차

### 0. 기획안 로드
- 인자로 받은 기획안 파일을 Read로 읽는다
- 작업 항목(`- [ ]` 체크리스트)을 추출하여 TaskCreate로 등록
- 기획안 파일 경로를 기억한다 (매 단계마다 재참조)

### 1. 작업 루프 (각 TODO 항목마다 반복)

#### 1-1. 작업 시작
- TaskUpdate로 해당 작업을 `in_progress`로 변경
- 필요한 파일을 Read로 읽고 작업 수행

#### 1-2. 작업 완료 후 자가 검수
작업이 끝나면 **반드시 아래 3단계를 수행**:

**① 기획안 재확인**
- 기획안 파일을 Read로 다시 읽는다
- 현재 완료한 작업이 기획 의도와 일치하는지 확인
- 다음 작업이 무엇인지 파악

**② 변경 사항 검수**
- 수정한 파일을 다시 Read로 확인
- 의도하지 않은 변경이 없는지 확인
- 기존 기능이 깨지지 않았는지 확인

**③ 체크리스트 출력** (화면에 표시)
아래 형식으로 출력:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 검수 완료 | 기획 체크 완료 | 다음 작업 진행
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📋 [작업명] 완료 체크리스트:

변경 사항 (전→후):
  • 파일명: 변경 전 상태 → 변경 후 상태
  • 파일명: 변경 전 상태 → 변경 후 상태

진행률: [N/전체] 완료
다음 작업: [다음 TODO 항목]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 1-3. 검수 실패 시
- 문제점을 구체적으로 출력
- 즉시 수정 후 다시 검수
- 수정해도 안 되면 사용자에게 보고

#### 1-4. 다음 작업으로
- TaskUpdate로 현재 작업을 `completed`
- 기획안의 체크리스트에서 `- [ ]` → `- [x]`로 업데이트 (Edit)
- 다음 TODO 항목으로 진행

### 2. 전체 완료
모든 작업 항목 완료 후:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎯 전체 작업 완료
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 최종 전후 비교:
  • [각 변경 사항 전→후 요약]

📁 수정된 파일 목록:
  • 파일1 (신규/수정/삭제)
  • 파일2 (신규/수정/삭제)

⚠️ 주의사항:
  • [있으면 기록]

💡 후속 작업 제안:
  • [자연스러운 다음 단계 2~3개]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

- 기획안 파일의 모든 `- [ ]`가 `- [x]`인지 최종 확인
- "/작업보고" 생성을 안내

## 팀 모드 (대규모 작업 — 선택)

작업 항목이 **8개 이상**이거나 **수정 파일 5개 이상**이면, 여러 서브에이전트로 나눠 진행할 수 있다.
(초보자는 몰라도 됨 — Claude가 알아서 판단한다.)

### 팀 소집 기준
- TODO 항목 8개 이상 → 팀 모드
- 예상 수정 파일 5개 이상 → 팀 모드
- 아키텍처 변경 포함 → 팀 모드
- 단순 설정/문서 작업만 → 팀 불필요, 일반 모드 진행

### 팀 모드 역할 (Agent 도구로 위임)
1. **설계** (Plan 서브에이전트): 전체 설계안 — 작업 순서, 의존성, 병렬 가능 여부
2. **조사** (Explore 서브에이전트): 영향 범위 — 관련 파일, 의존성
3. **병렬 구현** (general-purpose 서브에이전트 여러 개): 독립적인 파일 수정을 동시에
4. **검수** (별도 서브에이전트): 전체 완료 후 기획안 전체 대조

## 중요 규칙
- **기획안을 매 작업마다 Read로 다시 읽어라** — 캐시하지 마라, 항상 최신 상태 참조
- **중단하지 마라** — "계속할까요?" 묻지 말고 끝까지 진행
- **검수 없이 다음 작업으로 넘어가지 마라** — 체크리스트 출력이 필수
- **전후 비교가 핵심** — "뭘 했다"가 아니라 "뭐가 바뀌었다"를 보여줘라
- **팀 모드는 복잡한 작업에만** — 단순 작업엔 오히려 비효율
